错误日志
===

错误日志（Error Log）用于记录 MySQL 启动、运行和停止过程中的错误、警告和重要事件。

### 启动和设置错误日志

错误日志通常默认启用，也可以在配置文件中指定日志位置。

例如：

```ini
log_error=/path/to/error.log
```

通过合理配置日志路径，可以更方便地集中管理和排查问题。

### 查看错误日志

错误日志通常可以直接通过系统命令查看，例如：

```bash
tail -f /path/to/error.log
```

常见查看场景包括：

- MySQL 启动失败
- 配置项写错
- 权限问题
- 崩溃或异常退出

### 删除错误日志

错误日志文件本质上是普通日志文件，可以结合日志轮转或手工管理。

不过删除前要确认：

1. 是否已经做好归档
2. 是否影响后续排障
3. 是否需要先重载或重启服务

### 小结

错误日志是排查 MySQL 运行问题的第一入口。数据库启动异常、配置错误、崩溃等问题，通常都应该先看错误日志。
