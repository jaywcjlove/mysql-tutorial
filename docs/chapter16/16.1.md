优化简介
===

性能优化不是单一技巧，而是一套系统性工作。它可能涉及 SQL 语句、索引设计、表结构、服务器参数和硬件资源。

### MySQL优化介绍

MySQL 优化的目标通常包括：

- 降低查询耗时
- 提高并发能力
- 减少资源消耗
- 提升系统稳定性

常见优化方向有：

- 优化查询语句
- 优化索引设计
- 优化数据库结构
- 优化服务器配置

### 优化的原则

做优化时，通常遵循以下原则：

1. 先定位瓶颈，再优化。
2. 先优化高频、慢速、关键路径上的问题。
3. 优先做收益高、风险低的优化。
4. 任何优化都应有验证结果，而不是只靠猜测。

### 小结

优化不是“把参数都调大”或者“把索引都加上”，而是基于实际瓶颈做有针对性的改进。
